1. Introducción

Este manual técnico tiene como propósito proporcionar la información necesaria para el entendimiento lógico, la instalación y estructuración del sistema "Plataforma de Finanzas Personales XP". Está dirigido a personal técnico y administradores que deseen proveer mantenimiento al código base y bases de datos NoSQL de Firebase. El sistema está construido usando el paradigma Serverless con integraciones de TailwindCSS para diseño acelerado de Glassmorphism.

2. Descripción y Arquitectura

El sistema "Plataforma de Finanzas" es un aplicativo web moderno de control de gastos, cálculo de deudas y reproducción de material educativo financiero. A diferencia de las arquitecturas antiguas, el proyecto no utiliza un servidor monolítico. Su frontend se compone de HTML5, Vanilla JavaScript (ES6+), y utilidades Tailwind CSS. Su backend se sostiene nativamente en la infraestructura de Google Firebase (Firestore Database y Authentication), facilitando operaciones CRUD a velocidad extrema y evitando administración de servidores.

3. Características de los usuarios del sistema

El sistema cuenta con un administrador genérico de permisos por medio de Firebase Auth. Principalmente, existen dos roles:

  • Usuario Estándar: Posee un sandbox privado donde su información no choca con ningún otro usuario. Las colecciones (deudas, movimientos, categorías) se relacionan exclusivamente mediante el identificador "user_email".
  • Usuario Administrador: Aquel que tiene acceso a la consola central de Google Firebase, habilitado para suspender cuentas, borrar colecciones masivas, o recuperar accesos maestros mediante la Authentication Console.

4. Requisitos de Hardware y de Software

Requisitos de Cliente (Hardware)

  • Procesador: Dual Core o superior (ARM o x86).
  • Memoria RAM: 2 GB recomendado (para procesar Canvas en Chart.js).
  • Pantalla: Compatible con entorno Responsive (Móvil, Tablet, Desktop).

Requisitos de Cliente (Software)

  • Sistema Operativo: Plataforma agnóstica (Windows, macOS, Linux, iOS, Android).
  • Navegador: Google Chrome 90+, Safari 14+, o Firefox Quantum.

5. Instrucciones de instalación y despliegue

Para establecer un entorno de desarrollo local y desplegar a producción la plataforma, deben seguirse los siguientes pasos:

  1. Descargar el repositorio de código fuente del sistema en formato .zip desde GitHub o hacer un `git clone https://github.com/juasanchez33-ai/aplicatorio-web.git`.
  2. Dentro del directorio `/app`, observarás páginas modulares de plantillas puras estáticas (HTML) sin enrutadores complejos.
  3. Para levantar la prueba local no se necesita Node.js ni compiladores robustos. Una simple extensión como "Live Server" de VSCode o `python -m http.server` es suficiente para revisar el sitio en el navegador (localhost:8000).

6. Módulos Técnicos y Firebase

La base de datos (Firestore) se organiza velozmente en Nodos (Collections), los cuales son dinámicos:

  • Nodos "movements": Contiene ingresos y egresos. Cada documento registra {amount, date, concept, user_email, type}.
  • Nodos "debts": Registra montos brutos totales y saldo pendiente {total_amount, paid_amount, name, due_date}.
  • Nodos "payments": Registro de transacciones hijas de abonos a deudas. Interconectada lógicamente con un disparador local en JavaScript (`app.js`).
  • Nodos "categories": Un registro estático modificado dinámicamente según el usuario que le asigna a cada tipo de gasto un {color, icon, name}.

7. Solución de problemas

Problema Causa probable Solución recomendada
La app reporta FirebaseError (auth/admin) Llaves caducadas o dominio bloqueado Acceder a Firebase Console -> Authentication -> Settings -> Authorized Domains, y añadir el localhost o url de Vercel/Render.
Datos no se guardan en el Dashboard Errores en sintaxis JSON de submit Revisar la consola F12 de Chromium. Verificar si `window.firebaseAddData` reporta fallos de permisos de lectura/escritura (reglas de Firestore).
Las gráficas no renderizan Chart.js no instanciado a tiempo Asegurarse de mantener `updateDashboardCharts()` protegido tras un DOMContentLoaded y chequeando que el `canvas` exista.
